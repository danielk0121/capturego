## 3. e2e 테스트 (수동 조작 필요)

e2e 테스트 파일: `app/core/e2e_test.go` (`//go:build e2e` 빌드 태그)

실행 명령:
```
go test ./core/... -tags e2e -v -run E2E
```

### 실행 조건

e2e 테스트는 아래 조건이 충족될 때만 실행 가능하다.

| 조건 | 설명 |
|------|------|
| macOS 화면 기록 권한 | 시스템 환경설정 > 개인 정보 보호 > 화면 기록에서 터미널 허용 필요 |
| macOS 손쉬운 사용 권한 | 핫키 등록에 필요 |
| 사용자 직접 조작 | `screencapture -i` 인터랙티브 모드로 영역 지정, 스크롤 등 수동 입력 필요 |

### e2e 테스트 목록 및 검증 방법

| 테스트 | 검증 내용 | 실행 방법 |
|--------|-----------|-----------|
| TestE2E_듀얼세이브_캡처_파일생성 | 영역 캡처 후 임시 디렉토리에 PNG 파일 생성 확인 | 실행 후 마우스로 영역 드래그 |
| TestE2E_스크롤_캡처_병합파일_생성 | 스크롤 캡처 후 병합 PNG 파일 생성 확인 | 실행 후 영역 지정 + 스크롤 |
| TestE2E_Nagware_임계값_도달시_알림 | CaptureCount=50 상태에서 macOS 알림 표시 확인 | 실행 후 알림 팝업 육안 확인 |
| TestE2E_설정_저장_경로_변경후_캡처_위치확인 | 경로 변경 전/후 각각 올바른 위치에 파일 생성 확인 | 실행 후 2회 영역 지정 |

### e2e 테스트 미실행 사유

현재 세션에서 e2e 테스트를 Claude Code가 자동으로 실행할 수 없는 이유:

1. **`screencapture -i`는 인터랙티브 GUI 모드**로, 사용자가 직접 마우스로 캡처 영역을 지정해야 한다. 자동화 환경에서는 영역 지정 없이 프로세스가 블로킹 상태로 대기하거나 취소된다.
2. e2e 테스트는 **실제 macOS 화면 기록 권한**이 부여된 터미널에서 실행해야 한다.

## 4. 로직 검증 (코드 리뷰 기반)

GUI 조작 없이 검증 가능한 핵심 로직을 코드 분석으로 확인하였다.

### DualSaveCapture 로직

- `screencapture -i -x <파일경로>` 실행 → 파일 존재 여부로 취소 감지
- 파일 저장 확인 + `osascript`로 클립보드 복사를 **고루틴 병렬 처리**
- 클립보드 실패 시 경고만 기록, 파일 저장은 유지 (올바른 방어적 처리)
- `IncrementCaptureCount()` 호출로 Nagware 카운트 누적

### ScrollCapture 로직

- 첫 프레임: `screencapture -i` 로 영역 지정
- 이후 프레임: `screencapture -R <x,y,w,h>` 로 동일 영역 300ms 간격 반복 촬영
- `framesIdentical()` 16×16 픽셀 샘플링으로 스크롤 종료 감지
- `StitchImages()` 로 겹침 제거 후 수직 병합 저장

### Nagware 로직

- `nagwareThreshold = 50` 도달 시 첫 알림
- 이후 `nagwareRepeatInterval = 20` 간격으로 반복
- `LicenseActivated` 또는 `NagwareDisabled` 플래그로 영구 비활성화

### StitchImages 유닛 테스트 커버리지

유닛 테스트가 핵심 알고리즘을 충분히 검증하고 있다.

| 시나리오 | 검증 |
|----------|------|
| 단일 이미지 | 높이 그대로 유지 확인 |
| 겹침 없는 두 이미지 | 합산 높이(80px) 확인 |
| 겹침 있는 두 이미지 | 병합 후 높이 < 합산 확인 |
| 빈 목록 입력 | 오류 반환 확인 |
| 동일/다른 이미지 판별 | framesIdentical 정확도 확인 |
