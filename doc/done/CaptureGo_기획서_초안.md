## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | CaptureGo (캡쳐고) |
| 제품 정의 | macOS 환경에서 스크린샷 캡처 시 파일 저장과 클립보드 복사를 동시에 수행하는 초경량 백그라운드 유틸리티 |
| 타겟 환경 | macOS (Apple Silicon 아키텍처 기반 시스템 최적화) |
| 핵심 가치 | 최소 시스템 리소스 / 즉각적인 듀얼 세이브 / 제로 서드파티 의존성 |

> **개발 철학:** 무거운 프론트엔드 프레임워크 및 불필요한 서드파티 라이브러리(OCR 등)를 철저히 배제하고, 순수 Go 언어와 네이티브 OS 기능을 활용해 극강의 실행 속도와 안정성을 확보한다.

## 2. 시장 조사 및 경쟁 분석

macOS 스크린샷 유틸리티 시장의 주요 경쟁 제품인 CleanShot X와 Shottr를 가격 정책, 기능, 타겟 유저 관점에서 분석하여 CaptureGo의 포지셔닝 근거를 도출합니다.

### 2-1. CleanShot X — 프리미엄 올인원 지향

| 항목 | 내용 |
|------|------|
| 가격 정책 | 유료 전용 모델 (1회성 단건 구매 또는 월간 구독) |
| 단건 구매 | $29 (평생 라이선스) — 구매 시 1년간 메이저/마이너 업데이트 포함 |
| 업데이트 연장 | 1년 이후 최신 버전 업데이트 시 연 $19 추가 비용 발생 |
| 구독 플랜 | 월 $8 ~ $10 (Pro 플랜) — 무제한 클라우드 스토리지, 커스텀 도메인, 팀 협업 기능 포함 |
| 무료 버전 | 없음 — 체험판 없이 결제 후 30일 환불 보장 정책만 제공 |

**핵심 기능 및 특징**

- 화면 녹화 (비디오 / GIF 출력 모두 지원)
- 화면 위에 캡처 이미지를 핀(Pin) 형태로 고정하는 기능
- 캡처 후 전문적인 배경 자동 추가 (Presentation-ready 이미지 생성)
- 자체 클라우드 업로드 후 공유 링크 자동 클립보드 복사
- '공유와 워크플로우 최적화'에 집중된 올인원 솔루션

### 2-2. Shottr — 가볍고 빠른 로컬 유틸리티 지향

| 항목 | 내용 |
|------|------|
| 가격 정책 | 부분 유료화 (Freemium + Nagware 모델) |
| 단건 구매 | $8 Basic 티어 / $30 Friends Club 티어 — 결제 시 팝업 영구 제거 |
| 구독 플랜 | 없음 — 철저히 로컬 기반, 클라우드 연동 없음 |
| 무료 버전 | 제공 — 기간 제한 없이 핵심 기능 무료 사용 가능 |
| 무료 제한 | ① 주기적인 결제 유도 Nagware 팝업 ② 커스텀 배경색 등 일부 꾸미기 기능 제한 |

**핵심 기능 및 특징**

- 네이티브 수준의 압도적인 캡처 속도
- 긴 웹페이지를 하나의 이미지로 이어 붙이는 스크롤 캡처
- 이미지 내 텍스트를 인식하는 OCR 기능
- 픽셀 단위 측정을 위한 디자이너용 눈금자(Ruler) 기능
- '개인의 작업 생산성' 향상에 집중된 로컬 유틸리티

### 2-3. 경쟁사 비교 요약

| 항목 | CleanShot X | Shottr |
|------|-------------|--------|

| 단건 구매 | $29 (평생 라이선스) | $8 Basic / $30 Friends Club |
| 업데이트 | 구매 후 1년, 이후 연 $19 | 구매 후 지속 업데이트 |

| 핵심 강점 | 화면 녹화, GIF, 클라우드 공유 워크플로우 | 압도적 속도, 스크롤 캡처, OCR, 픽셀 눈금자 |
| 타겟 유저 | 팀 협업 / 콘텐츠 크리에이터 | 개인 개발자 / 디자이너 |

### 2-4. CaptureGo 포지셔닝 분석

두 경쟁 제품 사이에서 CaptureGo가 점유할 수 있는 차별화된 포지션을 도출합니다.

| CleanShot X | ★ CaptureGo 목표 포지션 | Shottr |
|-------------|------------------------|--------|
| 올인원 / 프리미엄 / 팀 협업 | 초경량 / 듀얼 세이브 / Nagware 수익화 | 경량 / 로컬 집중 / 개인 생산성 |
| $29 (1회) ~ $10/월 | 무료 + 소액 팝업 해제 | $8 ~ $30 (1회) |

> **차별점 요약:** CleanShot X가 채우지 못하는 '무료 진입 장벽 없음'과 Shottr가 제공하지 않는 '파일 저장 + 클립보드 동시 처리(듀얼 세이브)'를 결합. 빠르고 가볍되 결과물이 두 곳에 동시 저장되는 유일한 포지션을 점유한다.

경쟁사 분석을 통해 도출된 CaptureGo 전략 방향:

- **진입 문턱 제거:** 무료 버전 제공으로 즉각적인 유저 확보 (Shottr 전략 차용)
- **핵심 기능 집중:** OCR·클라우드 등 부가 기능 제외, 듀얼 세이브 단일 가치에 집중
- **Nagware 수익화:** 과도하지 않은 알림으로 자연스러운 유료 전환 유도 (Shottr 모델 참고)
- **초경량 철학:** 순수 Go + 네이티브 API로 CleanShot X 대비 압도적인 리소스 효율성 확보

## 3. 기술 스택 및 아키텍처

### 3-1. 기술 스택

| 구분 | 기술 / 라이브러리 | 역할 및 비고 |
|------|-----------------|-------------|
| Core & Backend | Golang | 동시성 처리, 시스템 API 제어 |
| Local Web Server | Gin Framework | 경량 API 및 정적 파일 서빙 |
| Frontend (설정 UI) | Vanilla JS / CSS / HTML5 | go:embed 바이너리 내장, 별도 빌드 불필요 |
| 시스템 트레이 | getlantern/systray | macOS 상단 메뉴바 아이콘 제어 |
| 글로벌 단축키 | golang.design/x/hotkey | OS 전역 단축키 감지 |
| 캡처 엔진 | macOS screencapture CLI | macOS 내장 네이티브 캡처 도구 |
| 클립보드 제어 | macOS osascript (AppleScript) | 클립보드 이미지 적재 |
| 이미지 처리 | Go 표준 image 패키지 | 스크롤 캡처 병합(Stitching) 알고리즘 |

### 3-2. 시스템 디렉토리 구조

각 도메인의 역할을 명확히 분리하여 유지보수성을 높이는 모듈형 아키텍처를 채택합니다.

| 디렉토리 | 역할 |
|---------|------|
| `cmd/capturego/` | 애플리케이션 진입점. 시스템 트레이와 백그라운드 Gin 서버 초기화 및 실행 |
| `internal/hotkey/` | OS 전역 단축키 입력 감지 및 캡처 이벤트 트리거 |
| `internal/tray/` | macOS 상단 메뉴바 아이콘 및 드롭다운 메뉴 구성 |
| `internal/capture/` | 핵심 비즈니스 로직. 단건 캡처, 스크롤 캡처, 파일 저장, 클립보드 적재 |
| `internal/server/` | Gin 라우터 및 REST API 핸들러. 프론트엔드와 통신하여 설정값 관리 |
| `web/` | 앱 내 설정 화면 구성 순수 웹 자원 (go:embed로 패키징) |

## 4. 핵심 기능 요구사항

### F1. 듀얼 세이브 캡처 (단축키 기반)

- 글로벌 단축키 입력 시 macOS 네이티브 캡처 도구가 대화형(영역 지정) 모드로 실행됨
- 영역 지정 완료 후 Go Goroutine을 통한 두 작업 병렬 처리
- **[저장]** 사용자 지정 로컬 디렉토리에 고해상도 이미지 파일로 즉시 저장
- **[복사]** 저장된 이미지 데이터를 macOS 클립보드에 복사 → 즉각적인 `Cmd+V` 붙여넣기 지원

### F2. 스크롤 캡처 (Scroll Capture)

- 긴 웹페이지나 소스 코드를 하나의 이미지로 캡처하는 모드
- 스크롤 캡처 모드 활성화 후 영역 지정 → 스크롤 시 짧은 간격으로 연속 촬영 진행
- 촬영 완료 후 이미지 간 겹치는 픽셀 영역을 자체 알고리즘으로 계산 및 절삭
- 처리된 이미지를 하나의 긴 세로 이미지로 병합(Stitching)하여 저장

### F3. 로컬 웹 기반 설정 인터페이스

- 트레이 메뉴의 '설정 열기' 클릭 시 Gin 서버(`localhost`)로 OS 기본 브라우저 연결
- 캡처 파일 저장 기본 디렉토리 경로 지정
- 글로벌 단축키 커스터마이징
- 라이선스 키 입력 및 검증

### F4. 시스템 트레이 제어

- 앱은 Dock에 표시되지 않고 상단 메뉴바에만 상주
- 메뉴 항목: 캡처 시작 / 스크롤 캡처 시작 / 설정 열기 / ☕ 개발자 응원하기 / 앱 종료

### F5. 부분 유료화 플로우 (Nagware)

- 기본 기능은 모두 무료로 제공
- 누적 캡처 횟수 도달 또는 앱 구동 시 macOS 네이티브 알림으로 '커피 사주기' 안내 팝업 노출
- 알림 클릭 시 결제/후원 페이지로 이동
- 라이선스 키 입력 및 검증 완료 시 팝업 로직 영구 비활성화

## 5. 사용자 시나리오 (User Flow)

| 단계 | 상황 | 행동 및 결과 |
|------|------|-------------|
| ① | 초기 설정 | 앱 실행 → 트레이 아이콘 → '설정 열기' → 브라우저에서 캡처 저장 폴더 지정 및 저장 |
| ② | 일상 캡처 | 웹 서핑/코딩 중 글로벌 단축키 입력 → 마우스로 원하는 영역 드래그 |
| ③ | 결과 확인 | '찰칵' 소리와 함께 지정 폴더에 파일 자동 생성 + `Cmd+V`로 즉시 붙여넣기 가능 |
| ④ | 스크롤 캡처 | 긴 페이지 캡처 필요 시 트레이 메뉴 → 스크롤 캡처 모드 → 자동 병합 후 저장 |
| ⑤ | 결제 유도 | 일정 캡처 횟수 달성 시 OS 알림으로 후원 요청 → 무시 후 계속 사용 또는 후원으로 팝업 제거 |
